## Issues reported and resolved

### Issue 1 — `Authentication failed` on every poll
**Root cause:** The password the user typed during `email-read add` simply did not match the mailbox password. Verified by:
1. Decoding `passwordB64` (`OH1dNiNOciZnTy1CVnY4aw==`) → plaintext `8}]6#Nr&gO-BVv8k`.
2. Manually testing that exact plaintext via `openssl s_client -connect mail.attobondcleaning.store:993 -crlf -quiet` then `a1 LOGIN ...` → server returned `a1 NO [AUTHENTICATIONFAILED]`.
3. Therefore the CLI was faithfully sending the configured password; the password itself was wrong.

**Resolution path given to user:** reset password in cPanel (or use a simple test password), verify with raw openssl IMAP login first, then `email-read remove <alias>` + `email-read add` to update.

**Confirmed fix:** WatchState eventually showed `ab|1|` → IMAP login + INBOX select working.

### Issue 2 — "Watcher prints nothing after baseline"
**Root cause:** The watcher was *too quiet* on success. The original `pollOnce` only logged on errors, baseline, or new mail. A healthy idle poll (no new mail) emitted zero output, making it indistinguishable from a frozen process.

Secondary likely cause for "no email arrives": test emails sent from external accounts (e.g. Gmail) may never reach the mailbox at all (MX issues, spam folder, or the address simply not being deliverable). The watcher correctly reports nothing because there is nothing to fetch.

**Fix applied this session:** Added verbose per-poll logging — see "Code changes" below.

## Code changes

### `internal/mailclient/mailclient.go`
- Added `MailboxStats` struct (Name, Messages, Recent, Unseen, UidNext, UidValidity).
- Changed `SelectInbox()` signature from `(uint32, error)` to `(MailboxStats, error)`. Single caller (watcher) updated.

### `internal/watcher/watcher.go`
- Rewrote `pollOnce` to log every step:
  - `poll start`
  - `watch state loaded: lastUid=N lastSubject=...`
  - `dialing IMAP host:port (tls=...) as user`
  - `connected and logged in (took XXms)`
  - `mailbox "INBOX" stats: messages=N recent=N unseen=N uidNext=N uidValidity=N`
  - `fetching messages with UID > N (server uidNext=M)`
  - `no new messages (poll completed in XXms)` OR `fetched N new message(s)`
  - Per-message: `saved uid=N from=... subj=... file=...` or `duplicate uid=N`
  - Per-rule: `uid=N matched K rule URL(s)`, plus open/skip lines
  - `poll complete: processed=N newLastUid=N total=XXms`
- This makes the cause of any silence obvious from logs alone.
