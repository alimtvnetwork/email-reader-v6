# Selection Bar & Line Interaction — Code Block System

**Version:** 3.1.0  
**Updated:** 2026-04-16

---

## Overview

The selection bar and line interaction system provides IDE-like line selection within code blocks. Users can click, Shift+click, drag, and use keyboard arrows to select lines. A floating selection bar appears at the bottom of the code block showing the active selection and offering copy/clear actions.

**Source files:**
- `codeBlockLineHandlers.ts` — click, keyboard, pin/range logic
- `codeBlockDragHandlers.ts` — drag-select and hover-highlight
- `codeBlockActionHandlers.ts` — copy-selected and clear-selected handlers
- `codeBlockBuilder.ts` — HTML generation for the selection bar
- `useCodeBlockEvents.ts` — event delegation wiring

## State Management

### Ref-Based State

All selection state is held in React refs (not re-render state) for performance:

| Ref | Type | Purpose |
|-----|------|---------|
| `activeWrapperRef` | `Element or null` | The wrapper that currently owns keyboard focus |
| `anchorIdxRef` | `number` | The anchor line index for range selections |
| `cursorIdxRef` | `number` | The current cursor position (may differ from anchor during ranges) |
| `lastPinnedRef` | `Map<Element, number>` | Maps each wrapper to its last-pinned line index |
| `dragStateRef` | `DragState or null` | Active drag session (wrapper + anchor index) |

### PinRefs Interface

```typescript
interface PinRefs {
  activeWrapperRef: React.MutableRefObject<Element | null>;
  anchorIdxRef: React.MutableRefObject<number>;
  cursorIdxRef: React.MutableRefObject<number>;
  lastPinned: Map<Element, number>;
}
```

## Line Selection Methods

### Single Click

**Trigger:** Click on `.code-line` or `.code-line-number` (excluding `.code-tool-btn`, `.code-font-controls`, `.copy-selected-bar`)

**Flow:**
1. Resolve the wrapper via `.closest(".code-block-wrapper")`
2. Determine line index via `getLineIndex()` helper
3. Call `pinSingle()`:
   - Clear all `.line-pinned` classes in the wrapper
   - Add `.line-pinned` to the clicked line and its corresponding line number
   - Store the wrapper as active, set anchor and cursor to the clicked index
   - Update the selection bar label

### Shift+Click (Range)

**Trigger:** Click with Shift key held, when the wrapper has a previous pin

**Flow:**
1. Retrieve the previously pinned index from `lastPinned` map
2. Set anchor to previous pin, cursor to clicked line
3. Call `pinRange()`:
   - Clear all existing pins
   - Apply `.line-pinned` to every line from `min(anchor, cursor)` to `max(anchor, cursor)`
   - Update `lastPinned` map and selection bar

### Keyboard Navigation

**Trigger:** Arrow Up / Arrow Down when `activeWrapperRef` is set

**Guards:**
- Ignored if `document.activeElement` is `INPUT`, `TEXTAREA`, or `contentEditable`
- `e.preventDefault()` is called to suppress default scroll

**Without Shift:**
- Move the single pin to the adjacent line
- Scroll the new line into view via `scrollIntoView({ block: "nearest" })`

**With Shift:**
- Extend or contract the range from the anchor
- Anchor stays fixed; only cursor moves
- Selection bar updates to show the new range

### Drag Selection

**Trigger:** `mousedown` on a `.code-line-number` element

**Drag lifecycle:**
1. `mousedown` — Record anchor index, clear existing pins, pin the anchor line
2. `mousemove` (on `document`) — Use `document.elementFromPoint(clientX, clientY)` to find the element under the cursor. If it resolves to a line in the same wrapper, call `pinRange()` from anchor to current
3. `mouseup` (on `document`) — Clear drag state

**Resolution logic (`resolveDragIdx`):**
- First checks for `.code-line-number` via `.closest()` and uses DOM index
- Falls back to `.code-line` and reads `data-line` attribute (1-based, converted to 0-based)

## Hover Highlight

**Trigger:** `mouseover` on any element inside `.code-block-wrapper`

**Behavior:**
1. Resolve line index from target via `getLineIndex()`
2. Add `.line-highlight` class to the line and its line number
3. On `mouseout`, remove all `.line-highlight` classes in the wrapper

**Styling:**
- Line background: `hsl(220 15% 16%)`
- Line number color: `hsl(var(--primary))`
- Line number background: `hsl(220 15% 12%)`

## Selection Bar HTML

Generated by `buildSelectionBarHtml()` in `codeBlockBuilder.ts`:

```html
<div class="copy-selected-bar" style="display:none">
  <span class="copy-selected-label"></span>
  <button class="code-tool-btn copy-selected-btn" title="Copy selected lines">
    <svg class="copy-icon">...</svg>
    <svg class="check-icon" style="display:none">...</svg>
    <span class="copy-label">Copy selected</span>
  </button>
  <button class="code-tool-btn clear-selected-btn" title="Clear selection">✕</button>
</div>
```

The bar is rendered inside every `.code-block-wrapper`, after the code body.

## Header Selection Label

A secondary indicator in the code block header:

```html
<span class="code-selection-label" style="display:none"></span>
```

Placed in `.code-header-right`, next to the line count. Shows the same text as the selection bar label (e.g., "Lines 3-7").

## Selection Bar Actions

### Update Label (`updateSelectedBar`)

Called after every pin operation:
1. Query all `.code-line.line-pinned` elements in the wrapper
2. If none: hide both the bar and header label
3. If pinned lines exist: extract `data-line` values, compute min/max
4. Format label: "Line {N}" for single line, "Lines {from}-{to}" for range
5. Set `.copy-selected-label` text and show the bar (`display: flex`)
6. Set `.code-selection-label` text and show it (`display: inline`)

### Copy Selected (`createCopySelectedHandler`)

**Trigger:** Click `.copy-selected-btn`

**Flow:**
1. Find the parent `.code-block-wrapper`
2. Query all `.code-line.line-pinned` elements
3. Extract `textContent` from each, join with newlines
4. Copy to clipboard via `copyTextToClipboard()`
5. Show "Copied!" feedback using the standard icon-swap pattern (2000ms delay)

### Clear Selection (`createClearSelectedHandler`)

**Trigger:** Click `.clear-selected-btn` (the ✕ button)

**Flow:**
1. Find the parent `.code-block-wrapper`
2. Remove `.line-pinned` from all elements
3. Hide the selection bar (`display: none`)
4. Hide the header selection label (`display: none`)
5. Delete the wrapper from `lastPinned` map
6. If `activeWrapperRef` points to this wrapper, set it to `null`

## Event Delegation Wiring

All handlers are registered in `useCodeBlockEvents.ts` via event delegation on the markdown container:

| Event | Target | Handler |
|-------|--------|---------|
| `click` | container | `lineClick`, `copySelected`, `clearSelected` |
| `mousedown` | container | `dragStart` |
| `mouseover` | container | `lineHover` |
| `mouseout` | container | `lineLeave` |
| `mousemove` | `document` | `dragMove` |
| `mouseup` | `document` | `dragEnd` |
| `keydown` | `document` | `lineKey` |

Handlers are created fresh on each `useEffect` cycle (triggered by `html` or `setFullscreenBlock` changes) and cleaned up on unmount.
